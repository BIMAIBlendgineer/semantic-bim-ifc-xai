# Validation Delta: v0.1-public-sample20 to v0.1.1-public-validation

## Purpose

This note documents the validation changes between the first public sample snapshot and the public validation cleanup release.

## v0.1-public-sample20

`v0.1-public-sample20` introduced the public `sample20` records and a minimal replay workflow.

The public validation layer at that stage demonstrated:

- JSONL loading;
- replay over 20 public records;
- basic schema-oriented checks;
- public/private boundary documentation.

## v0.1.1-public-validation

`v0.1.1-public-validation` strengthened the public validation layer.

The validator now checks:

- required public record keys;
- expected types;
- `ok is True`;
- `validation.ok is True`;
- non-empty `sample_id`;
- `parsed_output` object;
- `expected_output` object;
- `canonical_output` object;
- non-empty `canonical_output.evidence_trace`.

The replay harness now applies stricter public contract checks before reporting `REPLAY_OK`.

The release also introduced:

- `scripts/public_forbidden_scan.py`;
- public README coverage for folders;
- schema contract mapping;
- updated public evidence tables;
- terminology cleanup around human review and non-executable preview states;
- `.gitignore` protection for generated preview models.

## Interpretation

A `PASS` under `v0.1.1-public-validation` is stricter than a `PASS` under `v0.1-public-sample20`.

This does not mean that `sample20` became a larger dataset. It remains a minimal public reproducibility sample of 20 records.

This does not mean that a final benchmark, trained model, adapter, production system or certification workflow was added.
